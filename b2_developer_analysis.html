Section 01

B2 SDK 전체 구조

B2의 개발 환경은 3개의 SDK 패키지로 구성되며, 모두 유니트리 전 로봇에서 공유된다.

SDK 패키지 설명 언어
unitree_sdk2 CycloneDDS 기반 핵심 통신 SDK. Go2, B2, H1 등 전 로봇 공용 C++, Python
unitree_sdk2_python Python 전용 래퍼. pip install로 간편 설치 Python
unitree_ros2 ROS2 래퍼 패키지. Humble 공식 지원 C++/Python (ROS2)
핵심 포인트: SDK는 유니트리 전 로봇에서 공유된다. B2 전용 기능은 Sport API의 명령 ID와 파라미터 범위에서 차이가 난다. 즉, SDK 학습은 한 번이면 다른 로봇에도 적용 가능하다.
Section 02

High-Level Sport API — 21개 명령

별도의 보행 알고리즘 개발 없이 즉시 사용 가능한 고수준 명령. 순찰·점검의 핵심 기반이다.

이동 · 기본 제어

API ID 명령 설명 활용 시나리오
1001 Damp 전 관절 감쇠 (비상 정지) 비상 상황 즉시 정지
1002 BalanceStand 균형 잡힌 기립 자세 정지 상태에서 센서 수집
1003 StopMove 이동 즉시 정지 장애물 감지 시 긴급 정지
1004 StandUp 엎드린 상태 → 기립 전원 ON 후 초기 자세
1005 StandDown 기립 → 엎드림 충전 또는 대기 모드
1008 Move vx, vy, vyaw 속도 제어 자율주행 이동의 기본
1010 Sit 앉기 자세 대기 자세
1020 SwitchGait 보행 패턴 전환 (trot 등) 지형별 보행 모드 변경

경로 · 자율주행 (핵심)

API ID 명령 설명 활용 시나리오
1028 Euler Roll/Pitch/Yaw 자세 제어 경사면 보정, 카메라 각도
1035 BodyHeight 몸체 높이 조정 점검 높이 최적화
1037 TrajectoryFollow 경로점(waypoint) 순차 이동 순찰 경로 자동 실행
1038 ContinuousGait 연속 보행 유지 장거리 순찰
1045 SpeedLevel 이동 속도 단계 설정 순찰 속도 vs 이동 속도
1101 FrontJump 전방 점프 장애물 넘기

코드 예시 — Python 이동 제어

b2_move_example.py
import unitree_sdk2py as sdk

# 1. DDS 통신 채널 초기화
channel = sdk.ChannelFactory.Instance()
channel.Init(0, "eth0")

# 2. Sport Client 생성
sport_client = sdk.SportClient()
sport_client.SetTimeout(10.0)
sport_client.Init()

# 3. 이동 명령 (전진 0.3m/s)
sport_client.Move(0.3, 0.0, 0.0)

# 4. 경로점 이동 (TrajectoryFollow)
path_points = [...]  # x, y, yaw, vx, vy, vyaw 배열
sport_client.TrajectoryFollow(path_points)
Section 03

Low-Level 관절 제어

12개 관절을 500Hz로 직접 제어하는 고급 기능. 커스텀 보행 알고리즘 연구용.

관절 ID 위치 구성
0 — 2 전방 우측 다리 (FR) Hip, Thigh, Calf
3 — 5 전방 좌측 다리 (FL) Hip, Thigh, Calf
6 — 8 후방 우측 다리 (RR) Hip, Thigh, Calf
9 — 11 후방 좌측 다리 (RL) Hip, Thigh, Calf
사용 조건: MotionSwitcherClient.ReleaseMode() 호출 필수 (기본 Sport Mode 해제). PD 제어 게인(Kp, Kd) 설정 필요. 부적절한 제어값은 로봇 파손을 유발할 수 있다.

현 단계에서는 불필요

순찰·점검 솔루션은 High-Level Sport API만으로 충분하다. Low-Level 제어는 커스텀 보행 알고리즘(연구용) 또는 특수 동작 개발 시에만 필요하며, 현재 우리의 사업 목표와는 관계없다.

Section 04

센서 접근 및 ROS2 연동

B2에 탑재된 센서와 ROS2 생태계를 통한 확장 가능성.

탑재 센서 접근 방법

센서 접근 방법 데이터 활용
3D LiDAR IP 192.168.123.20
ROS2 /utlidar/cloud		 PointCloud2 SLAM, 장애물 감지, 3D 맵핑
IMU /lowstate 토픽 가속도, 자이로, 쿼터니언 자세 추정, 경사 감지
관절 인코더 /lowstate 토픽 각도, 속도, 토크 상태 모니터링
전방 카메라 별도 드라이버/IP RGB 영상 물체 인식, 이상 감지
풋 포스 /lowstate 토픽 발바닥 압력 지형 판단

ROS2 주요 토픽

토픽 타입 설명
/sportmodestate SportModeState 위치, 속도, 보행 상태 실시간 조회
/lowstate LowState 관절 상태, IMU, 배터리 정보
/utlidar/cloud PointCloud2 3D 포인트클라우드 (LiDAR)
/api/sport/request Request Sport API 명령 전송
ROS2 활용의 핵심 이점: Nav2(자율 주행), SLAM Toolbox(맵 생성), RViz2(시각화) 등 성숙한 오픈소스 패키지를 즉시 활용할 수 있다. 처음부터 만들 필요가 없다.

개발 로드맵 및 난이도

SDK 환경 구축부터 24시간 무인 운용까지, 8단계 개발 로드맵.

1
Phase 1 — 즉시 시작
SDK 환경 구축 + 기본 이동 제어
난이도 낮음 1~2주
Python, Linux, 네트워크 기초 지식만으로 시작 가능. SDK 설치 후 Move, StandUp 등 기본 명령 테스트.
2
Phase 2
LiDAR 데이터 수집 + ROS2 연동
난이도 낮음 2~3주
ROS2 Humble 환경 구축, /utlidar/cloud 토픽 수신, RViz2로 포인트클라우드 시각화.
3
Phase 3
SLAM 맵 생성 (실내 환경)
난이도 중간 3~4주
SLAM Toolbox + LiDAR로 실내 환경 2D/3D 맵 생성. SLAM 이론 및 ROS2 Nav Stack 이해 필요.
4
Phase 4
자율 주행 (경로 계획 + 장애물 회피)
난이도 중간 4~6주
Nav2 Costmap + 경로 알고리즘 통합. 맵 기반 자율 이동 구현.
5
Phase 5
순찰 시나리오 통합 (웨이포인트 순찰)
난이도 중간 3~4주
웨이포인트 설정, 순찰 스케줄링, 상태 머신 설계. 시스템 통합 + 프로토타입 완성.
6
Phase 6
AI 이상 감지 (카메라 영상 분석)
난이도 높음 6~8주
컴퓨터 비전 + 딥러닝 모델 개발. YOLO 등 비전 모델로 이상 상황 실시간 감지.
7
Phase 7
원격 관제 시스템
난이도 중간 4~6주
웹/앱 대시보드, 실시간 통신 (WebSocket), 다중 로봇 모니터링 UI 개발.
8
Phase 8 — 최종
24시간 무인 운용 (자동 충전)
난이도 높음 8~12주
충전 도크 하드웨어 설계 + 도킹 알고리즘 개발. 완전 무인 자율 운용 체계 구축.

현실적 평가

1~5단계 (SDK 활용 + 자율 순찰): ROS2 경험이 있는 로봇 SW 엔지니어 1~2명이면 3~4개월 내 프로토타입 가능. ROS2 오픈소스 패키지를 적극 활용하면 개발 기간을 단축할 수 있다.

6~8단계 (AI + 관제 + 무인화): 추가 전문 인력 필요 (AI, 웹 개발). 디스펙터 Cerebix 연동 시 AI/관제 부분의 개발 부담을 경감할 수 있다.

Conclusion

핵심 결론

B2 SDK 분석을 통해 도출한 4가지 핵심 결론.

B2 SDK 분석 결론

SDK는 "이동 플랫폼"으로서 충분하다. 그 위에 얹는 것이 우리의 부가가치다.

SDK가 제공하는 것

  • 21개 고수준 API로 즉시 이동 제어
  • 보행 알고리즘 자체 개발 불필요
  • LiDAR, IMU 등 센서 데이터 접근
  • ROS2 공식 지원 (Nav2, SLAM 활용)
  • C++/Python 모두 지원

우리가 만들어야 하는 것

  • SLAM 통합 + 자율 주행 로직
  • AI 기반 이상 감지
  • 원격 관제 시스템
  • 외장 센서 연동 (스피커, 조명, 4G 등)
  • 자동 충전 도크 (24시간 운용)
ROS2 생태계가 개발 부담을 줄여준다. Nav2, SLAM Toolbox 등 성숙한 패키지를 활용하면 처음부터 만들 필요가 없다. 디스펙터 Cerebix 협업이 의미 있는 이유: SDK가 제공하지 않는 AI 분석, 관제 시스템, 엣지 컴퓨팅 영역이 정확히 디스펙터의 역량과 겹친다.
Action Items

첫 번째 액션

즉시 실행 가능한 3가지 우선 액션.

  1. SDK 환경 구축 + 기본 이동 테스트 — B2로 즉시 시작 가능. Python SDK 설치 후 Move, StandUp 등 기본 명령 검증. 예상 1~2주.
  2. ROS2 연동 + LiDAR 데이터 수집 — ROS2 Humble 환경에서 /utlidar/cloud 토픽 수신 확인. SLAM 준비 단계. 예상 2~3주.
  3. ROS2 경험 있는 로봇 SW 엔지니어 확보 — 개발 속도의 핵심 변수. 1~2명 확보가 프로토타입 일정을 결정한다.
